# 食譜收藏系統 - 產品需求文件 (PRD)

## 1. 專案概述
- **背景與動機**：使用者經常從不同平台（網頁、社群媒體、手寫筆記）搜集各種食譜，但這些資訊四處散落，難以整合。當真正要下廚時，往往需要花很多時間去找當時儲藏的食譜，並且在烹飪過程中的單位換算與食材採買也常遇到困難。
- **目標用戶**：料理愛好者、經常下廚的家庭成員、以及需要工具輔助的新手。
- **核心價值主張**：提供一站式的食譜儲存、分類與下廚輔助方案，讓找食譜、準備食材到烹飪的過程變得輕鬆流暢。

## 2. 功能需求

以下列出系統的主要功能：

1. **萬用食譜抓取器**
   - 作為 **料理愛好者**，我希望 **能貼上各大食譜網頁的網址直接抓取所需內容（如食材清單、步驟）**，以便 **快速把網頁上的食譜存進我的專屬食譜庫，不需手動複製貼上。**

2. **智慧食材份量轉換**
   - 作為 **料理愛好者**，我希望 **能輸入欲烹調的份數（如原本 2 人份改成 4 人份），系統自動重新換算所有食材的重量與用量**，以便 **在備料時能夠準備正確的食材數量，避免浪費。**

3. **自動化採買清單**
   - 作為 **料理愛好者**，我希望 **勾選要準備的食譜後，系統能自動把所需的食材與數量彙整成一張採買清單**，以便 **方便我去超市或市場一次採購所有必需品而不遺漏。**

4. **互動式下廚模式**
   - 作為 **料理愛好者**，我希望 **在烹飪時可以開啟下廚模式放大每個步驟提示，並且可以透過簡單點擊或語音來切換下一步**，以便 **雙手沾滿食材時也能輕鬆看到指導，專心完成烹調。**

5. **庫存管理與建議**
   - 作為 **料理愛好者**，我希望 **能記錄家中現有的食材，並透過系統推薦可以使用這些庫存變出的食譜**，以便 **快速決定今晚吃什麼，並順利消耗掉冰箱裡的即期食材。**

## 3. 非功能需求
- **技術限制**：
  - 框架：Python 的 Flask 框架
  - 模板引擎：Jinja2
  - 資料庫：SQLite (適合初期迅速開發與展示)
- **效能與安全考量**：
  - **效能**：萬用食譜抓取器的解析與儲存需在合理時間內（如 5 秒內）完成，不阻礙使用者操作。
  - **安全**：防範 XSS 攻擊，對來自外部網頁抓取的文字進行嚴格驗證與過濾。密碼等敏感資料需採用雜湊加密過後才能儲存。
  - **裝置適配度 (RWD)**：考量到「互動式下廚模式」和「採購清單」會高度依賴手機使用，網頁系統需有良好的行動端排版 (Mobile-First)。

## 4. MVP (Minimum Viable Product) 範圍
- **Must Have (必須有、核心價值)**：
  - 食譜的新增、編輯、刪除、與瀏覽功能。
  - 自動化採買清單（多張食譜的食材彙總統計）。
  - 智慧食材份量轉換（最基本的倍數乘除法運算）。
- **Should Have (應該有、重要功能)**：
  - 互動式下廚模式（包含步驟放大檢視與防止螢幕休眠機制）。
  - 萬用食譜抓取器（優先支援台灣最常見的 1~2 個大型食譜分享網站，如愛料理等）。
- **Nice to Have (有則加分、進階功能)**：
  - 庫存管理與建議（能手動輸入冰箱有的食材反向推薦現有食譜）。
  - 匯出功能或是分享連結給其他人。
